# UX Evaluation Checklist

変更・追加・レビュー時に使用する評価チェックリスト。  
すべての項目を通過してから実装を完成とする。

---

## A. 哲学整合チェック（全変更共通）

```
□ 空白・不足・未達成が画面のどこかに見えていないか
□ 「まだ足りない」と感じさせる数値・グラフ・バーがないか
□ ストリーク・連続日数・達成率の概念が混入していないか
□ カレンダーグリッド（空のマスが見えるUI）がないか
□ 花・記録・ポジティブな要素が「減る」ことがないか
□ テキストに「また明日も」「続けよう」的な強制感がないか
```

---

## B. ホーム画面チェック

```
□ 花が画面中央に大きく・主役として配置されているか
□ スクロールなしで1画面に収まっているか
□ 情報密度が低く「呼吸できる空間」があるか
□ 時間帯に応じた挨拶が表示されているか
□ CTA前に免罪符テキスト（「小さなことでいい」等）があるか
□ 今日記録済みの場合、「今日はもう書いたね ✦」が表示されているか
□ 花のキャプションがコピーガイドラインのトーンに沿っているか
□ 数値（記録数・%・分母）が見えていないか
```

---

## C. 記録画面チェック（EntryScreen）

```
□ プレースホルダーが「小さな日常の出来事」の例になっているか
□ 気分スタンプに「これだけでもOK」という説明があるか
□ 保存ボタンが hasContent === false のとき視覚的に非活性（opacity 0.38）か
□ 文字数カウンターが 43字未満のときグレー表示か（T.textMuted）
□ 文字数カウンターが 43字以上のときローズ表示か（T.rose）
□ ヒントテキストのトーンがプレッシャーを与えていないか
```

---

## D. 完了演出チェック（CompletionScreen）

```
□ Bloom に entries が渡されているか（stage 数値ではないか）
□ Particles（スパークル）が発火するか
□ 完了メッセージが DONE_MSG 配列からランダムに選ばれているか
□ メッセージに「また明日も」的なフォワードプレッシャーがないか
□ 「ホームに戻る」ボタンで home に遷移するか
```

---

## E. あゆみ画面チェック（AyumiScreen）

```
□ カレンダーグリッドが一切存在しないか
□ 空白日・記録していない日が可視化されていないか
□ 月ラベルが「文脈の区切り」として目立たず配置されているか（添え物）
□ サマリーカードの Bloom に entries が渡されているか
□ 「書くたびに、花びらが増えていく」などの加点フレーミングがあるか
□ 記録がゼロのとき「空白は罪じゃない」的なコピーが表示されるか
```

---

## F. 設定画面チェック（SettingsScreen）

```
□ 花（Bloom）がサマリーとして表示されているか
□ LightJar が存在しないか
□ 「瓶の満ち具合」など瓶に関する表現がないか
□ 記録の数が加点フレーミングで表示されているか（達成率・%なし）
□ エクスポートが機能するか（.txt ダウンロード）
□ 全削除確認ダイアログが出るか
```

---

## G. デザインシステム整合チェック

```
□ ハードコードカラー（#38e8ff 等）を使っていないか（T.* を使っているか）
□ 新しいアニメーション keyframes を追加していないか（既存の再利用）
□ フォントが Noto Serif JP / Zen Kaku Gothic New / Space Mono の範囲内か
□ 水平パディングが 24px で統一されているか（ホーム・あゆみ・設定）
□ ボーダーに T.border（rgba(100,200,255,0.16)）を使っているか
□ スプリングカーブ（cubic-bezier(0.34,1.56,0.64,1)）がインタラクション変形に使われているか
```

---

## H. コンポーネント整合チェック

```
□ Bloom が entries 配列を受け取っているか（全呼び出し箇所）
□ 廃止済みコンポーネントが参照されていないか:
   - LightJar
   - CalendarScreen 相当のもの
   - bloomStage() 関数
   - Bloom stage={数値} 形式の呼び出し
□ 新規エントリの id が `e_${Date.now()}` 形式か
□ localStorage の保存が entries/settings 変更時に useEffect で走るか
```

---

## 評価スコアリング（参考）

変更規模に応じて以下のカテゴリを必須チェック:

| 変更種別 | 必須カテゴリ |
|---|---|
| コピー変更のみ | A + 変更対象画面のカテゴリ |
| UI調整（既存画面） | A + B〜F（対象画面） + G |
| 新機能追加 | A + B〜H 全項目 |
| Bloomロジック変更 | A + D + E + H + `bloom-growth-model.md` の評価基準 |
